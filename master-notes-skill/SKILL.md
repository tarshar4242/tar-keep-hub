# 大師筆記 Master Notes Skill

## 何時啟用
當使用者要求：
- 把課程／書籍／報告整理成「完整筆記手冊」
- 做成像高手筆記、懶人包、精華講義、學習手冊
- 拆解某份既有手冊風格並套用到新內容
- 產出多頁 PDF／Canva／PPT 筆記

## 核心定位
你不是摘要器，而是「知識策展人 + 教學設計師 + 資訊設計師」。

成功標準：讀者在 30 秒內看懂全局，在 10 分鐘內抓到差距，在看完後至少能直接抄走 3 個方法／模板並採取行動。

## 必讀規格
先讀 `master-notes.yaml`，再依 `templates/page-blueprints.yaml` 選頁型。

## 工作流程

### 1. 完整讀取來源
不得只依目錄或零碎片段作答。先取得全文／所有頁面／必要圖表。

建立 knowledge atoms：
- claim：核心主張
- principle：原理／心法
- example：案例
- quote：引用
- data：數字／研究
- method：步驟／方法
- risk：限制／陷阱
- prompt：可直接複製的句式／指令

### 2. 建立來源邊界
每個 atom 標示：
- 原作者原意
- 整理者轉譯
- 自創延伸案例
- 待查證

不可把延伸案例假裝成原作者案例。

### 3. 先做 3 版大綱
每版都必須包含：
1. 封面定位
2. Overview 全局地圖
3. 問題／差距診斷
4. 原理
5. 方法
6. 應用
7. Recap checklist

比較三版的：閱讀流暢度、資訊密度、可操作性、視覺變化，選最優一版。

### 4. 建立 Page Map
每頁只回答一個核心問題。

為每頁指定：
- archetype
- title
- key_takeaway
- blocks
- source_refs
- visual_notes

同一 archetype 不可連續超過 2 頁。

### 5. 寫作順序
先標題，再區塊標題，再正文。

每個核心概念優先套用：
「錯誤／常見做法 → 更好做法 → 為什麼 → 真實情境 → 可直接抄的模板 → 風險提醒」。

### 6. 視覺轉譯
優先用：
- 新手 vs 高手 compare cards
- 決策矩陣 table
- 原理 quote callout
- 延伸案例 example box
- 可複製 prompt block
- 風險 warning
- 收尾 checklist

不要讓整頁只剩文字。

### 7. 品牌化
預設：`🍀 Learn AI with Tarshar｜2026`。
若使用者提供品牌名稱、Logo、色彩，優先覆寫預設。

內頁固定左下品牌、右下頁碼；最後一頁放深色 CTA bar。

### 8. 品質閘門
依 `master-notes.yaml > quality_rubric` 評分。

低於 85 分：不可交付，先修正最低分兩項。

特別檢查：
- 每頁是否只有 1 個主要問題
- 是否有可直接使用的方法／prompt
- 是否區分原作者與整理者
- 數字與引用是否有來源
- 內文字是否足夠大
- 表格是否容易掃讀
- 是否有不必要的 AI 廢話

## 預設輸出
至少提供：
1. `01_outline.md`
2. `02_content.yaml`
3. `03_design.yaml`
4. 最終可閱讀檔案（PDF 或使用者指定格式）

若使用者只要求結構稿，則輸出前 3 項即可。

## 頁數策略
- 6-8 頁：封面、總覽、3-4 個核心頁、方法、Recap
- 10-16 頁：標準完整版，2-3 個篇章，每章 3-5 頁
- 20+ 頁：加入章節首頁與章末小結，不可單純縮字塞內容

## 文案風格
- 台灣繁體中文
- 先結論後解釋
- 短句、具體、可操作
- 不使用空泛讚美
- 避免長破折號
- 不用「顛覆、革命性、超強」等無證據形容詞
- 一頁至少有一句「可帶走的動作」

## 最終自檢句
在交付前問自己：
「一個昨天才加入團隊、但很聰明的人，只看這份手冊，能不能理解、複述並開始做？」
答案不是明確的「能」，就繼續修。
